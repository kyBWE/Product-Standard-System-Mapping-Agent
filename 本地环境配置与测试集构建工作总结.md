### 2. LLM 跳过条件优化

在 `rag_match_engine.py` 中新增跳过条件（优先级最高）：

```python
if best.coarse_score >= 0.85:
    skip_llm = True
```

**效果**：粗召回融合分 ≥ 0.85 时直接跳过 LLM/Rerank，预计跳过率从 96.7% 提升至 98%+。

### 3. 测试集构建

#### 3.1 脚本设计

编写 `build_test_set.py`，流程如下：

1. 从 728,493 条产品中随机抽样 1,000 条（seed=42 可复现）
2. 四种引擎并行匹配（4 线程 ThreadPoolExecutor）
3. 投票确定 ground truth：
   - ≥ 3 种方式命中同一分类 → 直接采用（`voting_3/4` 或 `voting_4/4`）
   - 2:2 平票 → 调用 LLM 仲裁（`llm_arbitrate`）
   - 其他 → 取最高票（`top_vote_1`）
4. 输出 `output/test_set_1000.json`

#### 3.2 遇到的问题与修复

| 问题 | 原因 | 解决 |
|------|------|------|
| LLM 仲裁全部失败（604 条 `llm_arbitrate_fallback_top_vote`） | `max_tokens=10` 太小，DeepSeek-v4-flash 输出前有内部思考，10 token 不够返回实际内容（`finish_reason: length`） | `max_tokens` 从 10 改为 256；解析逻辑改为从回答中提取首个 A/B/C/D 字母 |

| 脚本无进度反馈 | 第一版统一写入，中途无法查看进度 | 新增每 50 条日志打印 + `test_set_progress.txt` 进度文件 |

## 三、关键发现

### 1. PageIndex 置信度的"假高"现象

PageIndex 强制 LLM 模式的置信度计算公式：

```
confidence = max(base, 0.45 + llm_ratio × 0.35)
```

LLM 每层都参与时 `llm_ratio = 1.0`，置信度可达 0.8+。但这是**局部最优**——LLM 逐层选择只看当前层兄弟节点，不感知全局，走错一层后续全错但置信度仍高。

**案例**："外泌体超级原料" → PageIndex Force 给出 id=5220（化妆品原料，置信度 0.92），RAG 给出 id=20094（生物活性肽，置信度 0.54）。LLM 仲裁选择了 20094，因为"外泌体"是生物技术概念而非化妆品。

**结论**：PageIndex 置信度反映"路径走的有多确定"，不代表"结果有多正确"。

### 2. 置信度计算机制

RAG 系最终置信度：

```
confidence = coarse_weight × coarse_score + fine_weight × fine_score
           = 0.4 × coarse_score + 0.6 × fine_score
```

- `coarse_score` = 0.6 × 语义相似度 + 0.4 × 文本相似度（trgm≥0.8 时偏移为 0.3×语义+0.7×文本）
- `fine_score`：LLM/Rerank 打分，或跳过时按规则给奖励分（同名1.0、文本≥0.8给1.0、包含关系0.7等）
