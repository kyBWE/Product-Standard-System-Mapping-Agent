## 一、当日业务模型（最终约定）

| 能力 | 数据来源 | 用途 |
|------|----------|------|
| **同义词扩展** | 批量匹配中的 **MATCHED** 结果 | 产品已挂到某分类，但品名 ≠ 类名 → 人工复核 → `/api/confirm` 写入 `syn_list` |
| **暂存箱** | **NO_MATCH / LOW_CONFIDENCE** | 仅暂存，不自动建节点 |
| **新节点扩展** | 暂存箱产品，**手动**点「生成新节点建议」 | PageIndex 定父节点 + 正式类名 → 人工批准/改名/拒绝 → 写入体系 |

要点：
- 同义词 **不能** 从未匹配产品硬加；
- 新节点建议 **不** 在批量匹配后自动生成（避免慢与幻觉干扰演示）；
- 栏位命名：同义词扩展 / 新节点扩展（替代原先易混的「体系扩展 / 体系扩展审核」）。

## 三、【重点】同义词扩展 — 筛选逻辑

**接口**：`POST /api/synonym/suggest_from_matches`  
**入口数据**：批量匹配里 `match_status == MATCHED` 且有 `matched_category_id` 的条目（未匹配不进同义词）。  
**前端默认（演示复核）**：`auto_strict=false`，`min_confidence=0`，`text_threshold=0.92`，`use_llm=false`。

### 3.1 设计意图（一句话）

「已经匹配到某正式分类，但**品名写法和类名不一样**」→ 才值得当同义词；  
品名≈类名（近重复）或太泛、已存在，则**不应**进扩展。

### 3.2 逐条硬过滤（按顺序，任一不通过即 skip）

```
MATCHED 条目
  │
  ├─① 状态必须是 MATCHED（NO_MATCH / LOW_CONFIDENCE 直接丢）
  ├─② confidence ≥ conf_th
  │     · 演示复核：conf_th = 0（只要 MATCHED）
  │     · auto_strict=true：conf_th ≈ 0.95（配置 syn_confidence_threshold）
  ├─③ 分类 ID 在 category_texts 中存在
  ├─④ 产品名 ≠ 分类名（完全相同没必要加同义词）
  ├─⑤ 产品名不在该类已有 syn_list
  ├─⑥ 产品名 ∉ GENERIC_SHORT_SYNONYMS（系统/设备/产品/装置…）
  ├─⑦ 字面相似度 text_sim < text_th   ←「近重复上限」
  │     · 指标：chinese_text_similarity（SequenceMatcher ∪ 字 bigram Jaccard ∪ 包含比）
  │     · 演示：text_th = 0.92；严格：≈ 0.65
  │     · 注意：pg_trgm 对纯中文常恒为 0，已弃用为过滤依据，仅作对照日志
  └─⑧ sanitize_syn_list 通过
        · ≤2 字：仅当与类名有包含关系才保留，否则拒绝
        · >2 字：保留
```

通过 ①–⑧ 的进入 `llm_candidates`（硬过滤通过集）。

### 3.3 可选软过滤

| 开关 | 默认 | 作用 |
|------|------|------|
| `use_vec` | false | 算产品名↔类名向量余弦，**只作参考分，不作硬砍**（真同义词向量也往往很高） |
| `use_llm` | **false**（演示） | `synonym_verification`：≥0.7 强候选；≥0.5 进人工；&lt;0.5 丢弃。打开会变慢且易滤光 |

演示导入关闭 LLM，是为了让 30 条测试集能看到同义词候选；生产可开 `use_llm=true` + `auto_strict=true`。

### 3.4 典型 skip 原因对照

| 情况 | 结果 | 原因 |
|------|------|------|
| 雾炮机未匹配 | 不进同义词 | ① 非 MATCHED → 应进暂存箱 |
| 「罗茨风机」匹配到「罗茨风机」 | skip | ④ 与分类名相同 |
| 「不锈钢球阀」vs「球阀」字面过像 | 可能 skip | ⑦ text_sim ≥ 阈值 |
| 「系统」「设备」作品名 | skip | ⑥ 泛词 |
| 「弯头」≤2 字且与类名无包含 | 可能 skip | ⑧ 清洗拒绝 |
| 高置信匹配且写法差异大 | **进待复核** | 硬过滤全过 |

### 3.5 人工确认后

批准走 `/api/confirm`：把产品名写入该类 `syn_list`（再次清洗），**不建新节点**。

## 五、【重点】新节点扩展 — 筛选 / 定位逻辑

**接口**：`POST /api/staging/prepare_new_nodes`（`method: pageindex`）  
**入口**：暂存箱 `status=pending` 的产品（或调试用 `product_names`）。  
**不做的事**：不再无约束让 LLM 自由编一整条路径；父节点必须来自真实树。

### 5.1 整体两段

```
暂存产品名
  │
  ├─ A. PageIndex 定父节点（只在已有树上走）
  │     根层强制选大类 → 逐层 enter 或 stop → 若干 promote 规则
  │
  └─ B. 在父节点下生成「正式类名」
        LLM suggest_formal_category_name → _formalize_new_category_name 规范化
        → 若正式类已存在则提示并入 / 调整父级
        → 写入扩展池待人工 ✅/✏/❌
```

### 5.2 A：父节点怎么选（含「筛掉不合理停层」）

函数：`_pageindex_locate_expansion_parent`

1. **根层**：必须选一个根大类（不允许 stop）；对不上名则 `layer_disambiguation` 强制选。
2. **中间层**（最多约 10 层）：对当前节点的子类名调用 `layer_pick_or_stop`  
   - 选中某子类 → **enter**  
   - 返回 stop / 对不上子类 → **停在当前节点作为父**
3. **子类过多（>40）**：先用 `chinese_text_similarity`（+包含加分）粗排 Top40，再交给 LLM（防 prompt 爆炸）。
4. **进入后立刻纠偏（硬规则）**  
   - 当前名以「其他」开头 → **promote** 回父节点并停止（避开兜底垃圾类）  
   - 当前名与产品名字面相似度 ≥ 0.85 → **promote** 回父并停止（避免把近同名细叶当父，导致再挂更细 SKU）

**筛选含义**：不是「过滤掉产品」，而是过滤掉「不适合当挂载父」的过细叶 / 其他兜底；父必须是树上真实节点。

### 5.3 B：拟建类名怎么筛（正式名，非 SKU）

1. LLM 在父路径 + 兄弟类名约束下提出 `new_node_name`。
2. `_formalize_new_category_name` 再筛：

| 规则 | 处理 |
|------|------|
| 空 / 等于父名 | 换候选或回退 |
| 等于产品名 | 过细 → 拒 |
| 以「T型」「304」等修饰开头 / 去修饰后仍过像产品 | 剥修饰（`_strip_product_modifiers`）再试 |
| 与产品 text_sim ≥ 0.92，或长度几乎相同且 ≥ 0.85 | 视为产品级叶子 → 拒 |
| 全部候选失败 | 回退 `其他{父名}` 或 `其他专用制品` |

3. **若规范化后的正式类名已在库中同名存在**：不重复建；尽量挂到其上位，并在 reason 里写「建议并入」。
4. 若最终类名仍等于产品名：置信度压到 ≤ 0.5，提醒人工改名。

### 5.4 与同义词的边界（筛选视角）

| | 同义词扩展 | 新节点扩展 |
|--|------------|------------|
| 前提 | 已 MATCHED 到某叶/类 | 匹配失败或低置信 |
| 核心筛 | 「写法不同但应对同一类」 | 「树上缺合适类，需新建正式子类」 |
| 禁止 | 近重复品名、泛词、已是 syn | 幻觉路径、SKU 细叶当类名、「其他」作父 |
| 产出 | 往**已有**类写 syn_list | 在**已有**父下新建类名（人工批） |

### 5.5 实测例子（说明筛选仍依赖 LLM 判断）

| 产品 | 建议路径 | 筛选是否合理 |
|------|----------|--------------|
| 雾炮机 | `…> 水资源专用机械 > 自来水生产专用设备 > 雾化喷洒设备` | 已避开植保；父类仍偏水利 |
| T型丝锥扳手 | `…> 模具 > 模具专用工具` | 正式名规则生效，但父支选错 |
| 黄饼 | `矿产品 > 黄金制品` | 停在矿产品对；类名被字面「黄」带偏 |

**耗时**：约 13 秒/条。  
**400 修复**：前端补 `body:'{}'`；后端 `get_json(silent=True)`。

### 5.6 算法演进（简述）

无约束自由路径（幻觉）→ 向量定父（近邻偏农业等）→ **PageIndex 定父 + 正式类名**（当日方案）。
